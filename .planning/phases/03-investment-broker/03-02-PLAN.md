**Objective section:** Describe what the command does — reads profile, analyzes portfolio allocation, assesses risk profile, recommends ETFs, suggests rebalancing, queries live prices.

**execution_context:** Load these via @path:
- `@~/.claude/finyx/references/disclaimer.md`
- `@.finyx/profile.json`

Do NOT load broker reference docs here — this command does not do broker comparison.

**Process phases (per RESEARCH.md Pattern 1):**

**Phase 1: Validation**
- Profile gate: `[ -f .finyx/profile.json ]` — error message directs to `/finyx:profile`
- Read profile, extract `countries.germany.brokers[*].holdings` and `countries.brazil.brokers[*].holdings`
- Detect which countries are active (same logic as finyx:tax)

**Phase 2: Holdings Check (per D-05, D-06, RESEARCH.md Pattern 3)**
- If holdings[] is empty across all brokers in all active countries:
  - Use AskUserQuestion to collect holdings interactively
  - Ask: broker name, then for each holding: ticker, shares, cost basis, asset class (equity-etf/bond-etf/reit-etf/stock/fii/fixed-income), geography (global/us/europe/emerging-markets/brazil/germany)
  - After collection, offer to save to profile.json via Write (mirror finyx:tax Phase 3.4 Sparerpauschbetrag broker collection pattern exactly)
- If holdings exist, proceed

**Phase 3: Portfolio Allocation — INVEST-01**
- Calculate total portfolio value (shares * current value — use cost_basis as proxy if no live data yet, note this in output)
- Break down by:
  - Asset class: sum by asset_class field
  - Geography: sum by geography field
  - By broker: sum by broker
- Output format: `FINYX > INVEST: PORTFOLIO ALLOCATION` banner, then tables (see RESEARCH.md "INVEST-01: Portfolio Allocation Table" code example)
- Show each breakdown as a Markdown table with Value, Allocation %, and Target % columns

**Phase 4: Risk Profile Assessment — INVEST-02**
- Check if `goals.risk_tolerance` is already set in profile. If set, use it and skip questionnaire.
- If not set, use AskUserQuestion for 5 questions (per RESEARCH.md "Risk Profile Assessment" section):
  1. Investment horizon: <3 years / 3-10 years / >10 years
  2. Portfolio drops 20% reaction: sell all / hold / buy more
  3. Primary goal: capital preservation / balanced growth / maximum growth
  4. Emergency fund: yes (3+ months) / no
  5. Income stability: stable / variable / retired
- Mapping logic (from RESEARCH.md):
  - >10yr + buy more + max growth + emergency fund = Aggressive
  - 3-10yr + hold + balanced + stable = Moderate
  - <3yr OR sell on drop OR no emergency fund = Conservative
- Output risk level with target allocation table
- Offer to save risk_tolerance to profile.json

**Phase 5: ETF Recommendations — INVEST-03**
- Based on risk profile from Phase 4, recommend ETFs using the canonical list from RESEARCH.md:
  - Conservative: 40-50% equity, 40-50% bonds → VWCE or IWDA (core) + bond ETF
  - Moderate: 70% equity, 20% bonds, 10% satellite → VWCE (core) + EIMI (satellite)
  - Aggressive: 90% equity, 10% satellite → VWCE (core) + EIMI + regional satellite
- For Germany-active users: include ISINs, TERs, note Teilfreistellung eligibility (>50% equity = 30% exemption), note accumulating vs distributing for Vorabpauschale implications
- For Brazil-active users: include B3-listed ETFs (BOVA11, IVVB11, XFIX11) with brief descriptions
- ETF selection criteria: Tracking Difference over TER, AUM >500M EUR preferred, Ireland-domiciled preferred, physical replication preferred (from RESEARCH.md)
- Output: `FINYX > INVEST: ETF RECOMMENDATIONS` banner, then recommendation table with ISIN, TER, key notes

**Phase 6: Rebalancing Suggestions — INVEST-04**
- Compare current allocation (Phase 3) against target allocation (Phase 4)
- Use 5 percentage point drift threshold per asset class (from RESEARCH.md, Betterment/Vanguard standard)
- For each asset class where abs(current% - target%) >= 5: emit rebalancing suggestion (buy/sell/hold)
- If no holdings or no drift >= 5pp: state portfolio is within tolerance
- Output: `FINYX > INVEST: REBALANCING` banner, table with Current %, Target %, Drift, Action columns

**Phase 7: Market Data Query — INVEST-05 (per D-04, D-08, D-09, D-10)**
- Ask user via AskUserQuestion: "Would you like to look up live market data for a specific ticker?"
- If yes, ask for ticker symbol
- Determine market: if ticker matches B3 pattern (4-letter + number, e.g., BOVA11, PETR4) use brapi.dev; otherwise use Finnhub
- **Finnhub path:**
  - Check `FINNHUB_API_KEY` env var. If unset: emit setup banner (see RESEARCH.md Pattern 5 code example), use WebSearch fallback
  - If set: `curl -s "https://finnhub.io/api/v1/quote?symbol=${TICKER}&token=${FINNHUB_API_KEY}"`
  - For European ETFs: append `.XETRA` suffix (e.g., VWCE -> VWCE.XETRA) per Pitfall 2
  - Parse response with `jq -r '.c'` if available, else `node -e` one-liner (see RESEARCH.md code example)
  - Display: current price (c), change (d), % change (dp), day high (h), day low (l)
- **brapi.dev path:**
  - Check `BRAPI_TOKEN` env var. If unset: emit setup banner, WebSearch fallback per Pitfall 3
  - If set: `curl -s "https://brapi.dev/api/quote/${TICKER}?token=${BRAPI_TOKEN}"`
  - Parse: `results[0].regularMarketPrice`
  - Display: price, change, % change
- Output: `FINYX > INVEST: MARKET DATA` banner with price table

**Phase 8: Disclaimer**
- Append full disclaimer.md content (same as finyx:tax Phase 6)

**Output banner format:** Use `━━━` separator lines with ` FINYX > INVEST: [SECTION] ` headers throughout, consistent with finyx:tax.

**Notes section:** Include:
- Holdings are read from profile.json. Update holdings by editing profile.json directly or re-running this command.
- Market data requires API keys (FINNHUB_API_KEY, BRAPI_TOKEN) — free registration at respective sites.
- Risk assessment can be re-run by clearing goals.risk_tolerance in profile.json.
- All recommendations are advisory only — see disclaimer.
