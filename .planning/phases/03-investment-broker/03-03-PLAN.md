**Process phases (per RESEARCH.md Pattern 1 for broker command):**

**Phase 1: Validation**
- Profile gate: `[ -f .finyx/profile.json ]` — same pattern as finyx:tax
- Read profile, detect active countries from `countries.germany.tax_class` and `countries.brazil.ir_regime`

**Phase 2: Reference Doc Staleness Check — per D-12, RESEARCH.md Pattern 4**
- For each loaded broker reference doc, extract `last_verified` from YAML frontmatter
- Use `node -e` for cross-platform date comparison (per Pitfall 5 from RESEARCH.md):
```bash
node -e "
  const last = new Date('LAST_VERIFIED_DATE');
  const now = new Date();
  const days = (now - last) / 86400000;
  if (days > 180) console.log('STALE');
  else console.log('OK');
"
```
- If STALE: emit staleness warning banner (see RESEARCH.md Pattern 4 banner format)
- Continue regardless (data is still useful, just flagged)

**Phase 3: German Broker Comparison — BROKER-01 (only if Germany active)**
- Output: `FINYX > BROKER: GERMAN BROKERS` banner
- Present fee comparison table from germany/brokers.md reference doc:
  - Columns: Broker, Trade Fee, Sparplan Fee, Custody, Regulated By
  - Rows: Trade Republic, Scalable Capital (FREE + PRIME+), ING, comdirect
- Include each broker's URL per D-14
- Highlight key differentiators: Trade Republic cheapest per-trade, Scalable best for PRIME+ high-volume, ING/comdirect for full-banking integration

**Phase 4: Brazilian Broker Comparison — BROKER-02 (only if Brazil active)**
- Output: `FINYX > BROKER: BRAZILIAN BROKERS` banner
- Present fee comparison table from brazil/brokers.md reference doc:
  - Columns: Broker, Corretagem, Custodia, Strengths
  - Rows: NuInvest, XP, BTG Pactual
- Include each broker's URL per D-14
- Note: B3/CBLC emolumentos are charged separately by the exchange, not by the broker

**Phase 5: Profile-Based Recommendation — BROKER-03**
- Use AskUserQuestion to collect (if not inferable from profile):
  1. Trading frequency: "How often do you trade? rarely (1-2x/month) / regularly (weekly) / actively (daily)"
  2. Primary strategy: "What's your primary approach? ETF savings plan (Sparplan) / buy-and-hold / active trading"
  3. Tax simplicity preference: "How important is automatic tax handling? very important / nice-to-have / I'll handle it"
- Decision matrix (from RESEARCH.md BROKER-03):
  - Sparplan + rarely + tax important → Trade Republic (DE) or NuInvest (BR)
  - Sparplan + any + wants full banking → ING (DE) or XP (BR)
  - Active + regularly/daily + tax important → Scalable PRIME+ (DE) or NuInvest (BR)
  - Active + cost-sensitive + willing to handle tax → mention Trading212/IBKR as alternatives but warn about manual Anlage KAP
  - Buy-and-hold + rarely → Trade Republic or NuInvest (lowest per-trade cost)
- Output: `FINYX > BROKER: RECOMMENDATION FOR YOUR PROFILE` banner (see RESEARCH.md BROKER-03 decision matrix pattern)
- Format: recommended broker, 2-3 reasons why, "choose if" conditions, "avoid if" conditions

**Phase 6: Tax Reporting Quality — BROKER-04**
- Output: `FINYX > BROKER: TAX REPORTING` banner
- For Germany-active users: table comparing German vs foreign broker tax handling:
  - Abgeltungssteuer withholding: auto (DE) vs manual (foreign)
  - Freistellungsauftrag: supported (DE) vs not possible (foreign)
  - Jahressteuerbescheinigung: generated (DE) vs not available (foreign)
  - Vorabpauschale handling: auto-deducted Jan (DE) vs manual Anlage KAP-INV (foreign)
- Explicit recommendation: "If tax simplicity matters, use a German broker as your primary. If you use a foreign broker, allocate your full Sparerpauschbetrag to your German broker(s)."
- For Brazil-active users: note that ALL Brazilian brokers require self-reported DARF for renda variavel gains. Informe de Rendimentos covers only income received. No tax-handling differentiation between BR brokers.

**Phase 7: Disclaimer**
- Append full disclaimer.md content
