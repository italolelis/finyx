**`<process>` section with 7 phases:**

**Phase 1: Read Profile Data**
- Read `.finyx/profile.json` and extract: `investor.income.total` (or `countries.germany.gross_income`), `investor.marginalRate`, `investor.children`, `investor.country`, `identity.family_status`, `identity.age` (or `investor.age`).

**Phase 2: GKV Breakdown (COST-01)**
Compute exact GKV monthly cost. Reference health-insurance.md Section 1.1 formulas — do NOT hardcode any rates.

For employees (Path 1):
```
monthly_gross_capped = min(gross_monthly, 5812.50)  # BBG monthly
gkv_employee = monthly_gross_capped x (0.073 + zusatzbeitrag_rate / 2)
```
- Use `zusatzbeitrag_rate = 0.029` (fallback from health-insurance.md Section 1.3) unless a specific fund rate is provided.
- PV rate: reference Section 1.4. With children: employee rate 1.9%. Childless 23+: 2.5%. Multi-child reduction per table in Section 1.4.
```
pv_employee = monthly_gross_capped x pv_employee_rate
total_employee_monthly = gkv_employee + pv_employee
```

For self-employed (Path 2):
```
gkv_self = monthly_gross_capped x (0.146 + zusatzbeitrag_rate)
pv_self = monthly_gross_capped x pv_full_rate
total_self = gkv_self + pv_self
```

Output in `<gkv_breakdown>` subsection with line-by-line formula showing base rate, Zusatzbeitrag, PV, employer share deducted (for employees), and final monthly total.

**Phase 3: PKV Estimate (COST-02)**
Compute PKV estimate from age and health risk tier. Reference health-insurance.md Section 2.1 (age brackets) and Section 4.2 (3-tier risk model).

```
# Get base bracket from Section 2.1 age table
pkv_base_low, pkv_base_high = age_bracket_range(profile.age)
pkv_base_mid = (pkv_base_low + pkv_base_high) / 2

# Compute risk tier from health flags (Section 4.2)
flag_count = sum(health_flags)
if flag_count == 0: surcharge = 0
elif flag_count <= 2 and no high-rejection flag: surcharge = 0.175  # midpoint of 10-25%
else: surcharge = 0.40  # midpoint of 30-50%

pkv_estimated = pkv_base_mid x (1 + surcharge)
```

- Check for high-rejection flags per Section 4.3 — if present, emit prominent warning that PKV is likely unavailable at standard rates.
- For employees: compute employer subsidy per Section 1.5 formula: `employer_subsidy = min((pkv_estimated + ppv) / 2, 613.22)`. Net PKV = pkv_estimated + ppv - employer_subsidy.
- PPV (private Pflegeversicherung) estimate: use ~€50-80/month benchmark.

Output in `<pkv_estimate>` subsection showing: age bracket, risk tier, surcharge, base estimate, employer subsidy (if employee), net PKV to employee.

**Phase 4: Family Impact (COST-03)**
Compare GKV Familienversicherung vs PKV per-person costs. Reference health-insurance.md Section 3.

- Read `identity.family_status` and `investor.children` from profile.
- GKV family: `total_family_gkv = total_employee_gkv` (partner and children free under Familienversicherung if partner income < EUR 603/month).
- PKV family: each member needs own policy. Use Section 3.2 indicative costs: partner EUR 300-600/month, child EUR 150-200/month.
```
pkv_family_total = pkv_member + pkv_partner + (num_children x pkv_child)
gkv_family_total = gkv_member_cost  # no multiplier
family_delta = pkv_family_total - gkv_family_total
```
- If single with no children: output "N/A — single household, no family impact comparison needed" and skip.

Output in `<family_impact>` subsection with side-by-side table (GKV vs PKV per family member) and monthly delta.

**Phase 5: Projection Table (ADV-01)**
Produce 10/20/30-year projections. Reference health-insurance.md Section 2.5 constants.

Three scenarios using these growth rates:
- Conservative: PKV +5%/yr, GKV +2%/yr (BBG proxy)
- Base: PKV +6%/yr, GKV +3%/yr
- Optimistic: PKV +3%/yr, GKV +4%/yr

```
For each scenario, for each horizon [10, 20, 30]:
  pkv_future = pkv_today x (1 + pkv_growth) ^ n
  gkv_future = gkv_today x (1 + gkv_growth) ^ n
  delta = pkv_future - gkv_future
  cumulative_pkv = sum(pkv_today x (1+r)^i for i in 0..n-1) x 12
  cumulative_gkv = sum(gkv_today x (1+r)^i for i in 0..n-1) x 12
```

- Calculate crossover year (when PKV becomes more expensive than GKV, if applicable) per specific idea from CONTEXT.md.
- Label each scenario clearly.
- Include non-linear growth warning from health-insurance.md Section 2.5.

Output in `<projection_table>` subsection with a table per scenario showing: Year 10/20/30 monthly cost, cumulative cost, delta, and crossover year.

**Phase 6: Tax Netting (ADV-03)**
Net PKV cost against section-10 EStG Basisabsicherung deduction. Reference health-insurance.md Section 5.

```
basisabsicherung_portion = pkv_monthly x 0.85
cap = 1900  # employees; 2800 for self-employed (Section 5.1/5.2)
annual_deductible = min(basisabsicherung_portion x 12, cap)
annual_tax_benefit = annual_deductible x marginal_rate
net_pkv_monthly = pkv_monthly - (annual_tax_benefit / 12)
```

Output in `<tax_netting>` subsection showing: gross PKV, Basisabsicherung portion, cap applied, annual tax benefit, net PKV after deduction.
