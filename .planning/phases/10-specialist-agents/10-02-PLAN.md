**`<process>` section with 5 phases:**

**Phase 1: Read Input Context**
- The orchestrating command passes the following in the Task prompt: user age, employment type (employee/self-employed), family status (single/married), number of children, gross income bracket.
- These parameters anchor all search queries (per D-04).
- Read health-insurance.md Section 2.1 for age bracket context and Section 2.3-2.4 for Beitragsruckerstattung and Selbstbeteiligung background.

**Phase 2: Search for PKV Providers**
Run WebSearch queries anchored to user profile. Use the following query templates:

Primary neutral source queries:
1. `"PKV Testsieger [current_year] Stiftung Warentest"` — per CONTEXT.md specific idea
2. `"PKV Vergleich [current_year] Finanztip beste private Krankenversicherung"`
3. `"PKV Tarife [employment_type] [age_bracket] krankenkasseninfo.de"`

Direct provider queries (fallback top-5 by market share per D-04):
4. `"[provider_name] PKV Tarif [current_year] Beitrag [age_bracket]"` — run for each of: Debeka, DKV, Signal Iduna, Allianz, HUK-COBURG

If generic queries return sparse results, fall back to direct provider lookups.

Use WebFetch to extract tariff details from promising search results. Prioritize pages with actual price tables or tariff calculators.

**Phase 3: Extract Provider Data**
For each provider found, extract:
- Provider name and tariff name
- Monthly premium for the user's age bracket and coverage level
- Beitragsruckerstattung policy: how many months refunded for claims-free year, conditions
- Selbstbeteiligung options: available deductible levels (EUR 300/600/1200/2000) and corresponding premium reduction
- Altersruckstellungen approach (if disclosed)
- Any 2024-2026 premium increase history (signals future trajectory)

Flag data freshness: note the publication date of each source. Mark anything older than 12 months as `[STALE SOURCE]`.

**Phase 4: Select Top 3 Providers**
Per D-03: compare exactly 3 providers — top recommendation + 2 alternatives. More creates analysis paralysis.

Selection criteria:
1. Premium competitiveness for user's age bracket
2. Beitragsruckerstattung generosity (months refunded)
3. Selbstbeteiligung flexibility (range of options)
4. Premium stability track record (lower recent increases preferred)
5. Market reputation from neutral sources

If fewer than 3 providers have sufficient data: include what is available, note gaps, flag `[LOW CONFIDENCE]`.

**Phase 5: Format Output**
Wrap entire output in `<insurance_research_result>` tags (per D-05). Structure:

1. Provider comparison table with columns: Provider, Tariff, Monthly Premium, Beitragsruckerstattung, Selbstbeteiligung Options, Recent Premium Trend
2. Top recommendation with 2-3 sentence rationale
3. Two alternatives with brief comparison notes
4. Per-provider Beitragsruckerstattung detail section
5. Per-provider Selbstbeteiligung pricing section
6. Sources list with URLs and publication dates
7. Confidence level + disclaimer reference

**Anti-patterns section:**
- Do NOT recommend a single "best" provider — present top 3 and let user decide
- Do NOT use Check24 as primary source — neutral sources first per D-02
- Do NOT fabricate tariff prices — if data unavailable, state "tariff data not found" with `[LOW CONFIDENCE]`
- Do NOT write any files — return structured output only
- Do NOT use Bash or Write tools
- Do NOT include health data in search queries — only age, employment type, family status

**`<output_format>` section:**
Provide the full XML template for `<insurance_research_result>` with provider comparison table, recommendation, alternatives, per-provider details, sources, and confidence/disclaimer — matching the structured output pattern from existing agents.
